Run Delivery Validation

Runs validation/validate_delivery.py against the uploaded JSONL via the hosted validation API. Set the API URL after deploying the backend service.

⏳ Heads up: this can take a while.

Large deliveries (300+ tasks) can take 10–15 minutes through the hosted API, since it downloads every environment ZIP on a shared server. For a much faster run, do it locally — there are no dependencies to install (just Python 3, plus curl and unzip, which macOS already has):

  1. Download the script: validate_delivery.py ↗ (view on GitHub)
  2. Open Terminal and cd to the folder where you saved it (e.g. cd ~/Downloads).
  3. Run the command below, swapping in the path to your delivery JSONL:
python3 validate_delivery.py --delivery "YOUR_DELIVERY.jsonl" --output report.json --workers 10

Click the command to copy it. It writes report.json with the same issues this tab analyzes. --workers 10 is a safe default; higher values download faster but can hit your machine's process limit on very large deliveries (it spawns curl/unzip per worker), so back off to ~10 if you see a BlockingIOError.
